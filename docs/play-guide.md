# AI村裁判 プレイガイド (約 5 分で読める)

ようこそ、監査官。ここは 6 人の AI 住民が暮らす小さな村。昨夜、住民の 1 人が襲われました。住民の中に紛れ込んだ **人狼が 1 人** います。あなたは村人ではなく、外から派遣された **監査官・裁判官** です。議論に直接加わることはできませんが、住民を尋問し、証拠を読み解き、裁判で人狼を裁くことができます。

関連ドキュメント: [README](../README.md) / [遊ぶ前の環境構築 (開発者向け)](dev-setup.md)

---

## 1. ゲームの目的

**3 日以内に、人狼を裁判で処刑できれば勝利です。**

- 村は AI 住民 6 人、うち人狼は 1 人。
- 1 日に使える尋問ポイントは **5**。夜が明けると回復します。
- 人狼を処刑できないまま 3 日目が終わる、村人が減りすぎる、あるいは誤処刑で村の信頼を失うと敗北です。

注意したいのは、**村人も合理的な理由で嘘をつく** こと。「自室にいた」と言い張る住民が、実は被害者と密会していただけ、ということもあります。怪しい = 人狼、とは限りません。

---

## 2. 1 日の流れ

毎日、次の順番でフェーズが進みます。

1. **朝** — 被害者・事件概要・その日の証拠 (3 枚程度) が提示されます。
2. **議論** — AI 住民たちが自動で議論します。発言には手がかり (と、人狼が仕込むミスリード) が含まれます。
3. **調査 (尋問)** — あなたが尋問ポイントを使って住民を問い詰めます。← ここがプレイヤーの主戦場。
4. **整理** — 気になる発言や証拠を **ピン留め** し、複数のピンを **矛盾候補** としてまとめます。
5. **裁判** — 容疑者 1 人を指名し、証拠と矛盾を突きつけて、処刑か保留かを決めます。
6. **夜 (監視)** — 翌朝の手がかりを得るため、夜に **監視対象を 1 人** 指定します。人狼はこの夜、誰かを襲撃します。
7. **翌朝** — 新しい被害・議論ログ・証拠・監視結果が追加され、次の日へ。

---

## 3. 尋問アクションとコスト

尋問は質問の種類ごとにポイントを消費します。1 日 **5 ポイント** をどう配分するかが勝負です。

| アクション | コスト | 用途                                               |
| ---------- | ------ | -------------------------------------------------- |
| 通常質問   | 1      | 基本的な聞き込み。アリバイや所感を尋ねる。         |
| 深掘り質問 | 2      | 行動を時系列で詳しく語らせる。嘘の綻びが出やすい。 |
| 証拠提示   | 1      | 手持ちの証拠を見せて反応を引き出す。               |
| 矛盾追及   | 2      | 過去の発言・証拠との食い違いを直接突きつける。     |
| 強制証言   | 3      | 強引に証言を引き出す。信頼度を下げやすい諸刃の剣。 |

ポイントが足りないアクションは実行できません。残りポイントは常に画面に表示されます。圧迫的な尋問は住民の **信頼度** を下げ、回答が非協力的になっていく点にも注意。

---

## 4. 証拠の 3 階層

証拠はすべて同じ重みではありません。内部的に 3 階層に分かれています。

- **確定証拠** — 単独では犯人を断定できないが、真犯人へ強く接続する手がかり (例: 深夜の扉ログ、靴跡)。
- **補助証拠** — 確定証拠を補強し、推理経路を強めるもの。
- **ノイズ証拠 (レッドヘリング)** — 村人を怪しく見せるが、最終的には別の理由で説明できるもの。

ここで効いてくるのが **「村人も合理的に嘘をつく」** 設計です。ノイズ証拠と村人の嘘が重なると、無実の住民が真犯人より怪しく見えることがあります。冤罪を避けるには、その怪しさが **人狼の犯行** で説明できるのか、それとも **別の秘密** で説明できるのかを見極める必要があります。

---

## 5. 裁判

確信が持てたら裁判へ。裁判では次を提示できます。

- **容疑者: 1 人** を指名
- **証拠: 最大 3 つ**
- **矛盾候補: 最大 2 つ** (整理フェーズでまとめたもの)

容疑者は **弁明** し、他の住民も反応・証言します。それを踏まえ、最終的にあなたが選ぶのは 2 択です。

- **処刑** — 容疑者を村から排除する。当たれば勝利、外せば村人を失い信頼度が下落。
- **保留** — その日は処刑しない。情報を集めてから翌日に賭ける。

人狼を処刑できれば、その瞬間に勝利です。

---

## 6. 勝敗とスコアランク

終了時には真相 (本当の人狼、夜間行動、各住民の秘密と嘘の理由、推理経路) がすべて開示され、あなたの推理と答え合わせができます。

プレイ結果は以下の観点 (人狼特定の成否 / 経過日数 / 誤処刑数 / 生存村人数 / 尋問ポイント効率 / 正しい矛盾指摘数 / 誤った矛盾指摘数 / 村の信頼度) で採点され、ランクが付きます。

| ランク | 評価         |
| ------ | ------------ |
| S      | 完全裁判     |
| A      | 優秀な監査官 |
| B      | 通常解決     |
| C      | 犠牲多数     |
| D      | 冤罪裁判     |

---

## 7. プレイのコツ

- **矛盾を探す。** 1 つ以上は「嘘を見抜ける矛盾」が必ず仕込まれています。発言と証拠の食い違いをピン留めして比較しましょう。
- **レッドヘリングに注意。** 一番怪しい人物が、たいてい人狼ではありません。その怪しさが別の秘密で説明できないか疑ってください。
- **ポイントを無駄打ちしない。** 序盤は通常質問で広く当たりを付け、確信に近づいてから深掘り・矛盾追及にポイントを集中させると効率的です。
- **強制証言は最後の手段。** 信頼度が崩壊すると AI が口を閉ざします。
- **監視対象を活かす。** 夜の監視は翌朝の手がかり量を大きく左右します。最も怪しい人物、または被害が及びそうな人物を選びましょう。

---

## 8. 例: 1 ターンの流れ

> **Day 2 の朝。** 証拠「深夜の扉ログ (23:52 に C の部屋の扉が開いた)」が提示される。
> **議論を読む。** C は「昨夜はずっと自室にいた」と主張。一方 A は妙に C をかばっている。
> **調査する。** まず C に _深掘り質問 (2pt)_ で夜の行動を語らせ、「外には出ていない」と引き出す。
> **矛盾を仕込む。** 扉ログと C の証言は食い違う。両者をピン留めして矛盾候補に。残り 3pt で A に _証拠提示 (1pt)_ し、A が C を守る理由 (実は A 自身の密会隠し) のヒントを得る。
> **裁判へ。** 容疑者を C に、証拠に「扉ログ」、矛盾候補に「C の在室主張 vs 扉ログ」を提示。C の弁明は破綻し、**処刑** を選択。
> **真相開示。** C が人狼、A の不審な行動はレッドヘリングだったと判明 — 見事 A の冤罪を回避。

それでは、よい裁きを。
